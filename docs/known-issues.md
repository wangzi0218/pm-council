## 结构性问题

- [ ] **设置入口分离** — LLM 配置 / NPC 管理 / 技能管理 应有清晰的分区或独立入口，当前全部堆在同一个面板里
- [ ] **空状态引导** — 没有活跃讨论时的空状态引导不够清晰，用户不知道该干什么

## Prompt & 对话质量

- [x] **括号内心活动** — NPC 回复中出现「（想了想）」「（皱眉）」等剧本杀风格 → 已加规则禁止
- [ ] **对话过于模板化** — NPC 回复有时像在完成任务，不像真实同事在讨论
- [ ] **LLM 沉浸人设 vs 延展话题** — NPC 应该在人设内自然回应，还是主动追问延展用户的话题
- [ ] **回复节奏调优** — 基于实际对话输出调整 prompt，持续迭代

## 交互流程

- [ ] **发起互动引导** — 用户打开应用后如何自然地开始第一轮讨论
- [ ] **新建讨论体验** — 点新建讨论后消息消失的体验问题（功能正确但感知不好）

## 产品决策

- [ ] **多角色交互模式** — 角色数量增多后，发言顺序、分组策略如何设计
- [ ] **多角色在一个对话 vs 多个对话** — 当前 3 NPC 在一个对话里，是否需要支持角色分组

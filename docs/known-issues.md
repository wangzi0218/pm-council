## Prompt & 对话质量

- [x] **括号内心活动** — NPC 回复中出现「（想了想）」「（皱眉）」等剧本杀风格 → 已加规则禁止
- [ ] **对话过于模板化** — NPC 回复有时像在完成任务，不像真实同事在讨论
- [ ] **LLM 沉浸人设 vs 延展话题** — NPC 应该在人设内自然回应，还是主动追问延展用户的话题

## 交互流程

- [ ] **发起互动引导** — 用户打开应用后如何自然地开始第一轮讨论
- [ ] **新建讨论体验** — 点新建讨论后消息消失的体验问题（功能正确但感知不好）
- [ ] **结构化交互** — 纯对话门槛高，需要快捷按钮降低互动成本：
  - 空状态话题引导卡片（「我有个新想法」「帮我评审方案」「拆解需求」）
  - 输入框上方快捷动作（「分析需求」「评审方案」「总结一下」）
  - 上下文感知建议（讨论发散时出「收敛一下」，NPC 发言后出「追问这一点」）

## 待做：记忆系统设计

NPC 应该有跨对话记忆能力，但需要界定边界：

| 应该记忆 | 不应该记忆 |
|----------|-----------|
| 项目背景、目标、约束 | 具体对话内容/原文 |
| 做过的决策和结论 | 每句话的原文 |
| 未解决的问题 | 谁说了什么 |
| 用户的偏好和风格 | 临时讨论细节 |

**实现方案：**
- 每轮对话结束后 LLM 自动生成 2-3 句摘要
- 摘要存入 `conversation_memory` 表（已建好）
- 下次讨论时，按相关性注入 NPC prompt
- 用户可以在群聊设置里查看和编辑摘要
- 未来可扩展为向量检索（DB 已预留 embedding 列）
